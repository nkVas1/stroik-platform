## Фаза 7 · Реальная аутентификация

**Цель:** заменить демо-логин на полноценную многоуровневую идентификацию.

### Сценарии

1. **SMS-OTP** (основной флоу для мобильных)
   - Провайдер: SMS.ru / SMS Aero (абстрактный интерфейс `SmsGateway`)
   - 4-значный код, TTL 5 мин, rate-limit 3 запроса/час на номер
   - Хранение хешей кодов, не plain-text

2. **Email + пароль** (опционально для PRO-аккаунтов)
   - `passlib` + bcrypt, политика паролей
   - Сброс через email-токен

3. **Госуслуги stub** (заглушка для будущей интеграции)
   - Кнопка-заглушка, документированный контракт OAuth-callback'а

### Уровни верификации

- **Level 1** — телефон подтверждён
- **Level 2** — паспорт загружен и прошёл OCR-валидацию
- **Level 3** — Госуслуги/ИНН/самозанятый статус подтверждены

### Безопасность

- Refresh-токены в HTTP-only cookie, access-токены — в памяти JS
- CSRF-токены на mutating-запросы
- Audit-лог входов (IP, user-agent, время)

## Фаза 8 · Маркетплейс v2

**Цель:** превратить базовый CRUD проектов в живой маркетплейс с реальными сделками.

### Поиск и навигация

- Полнотекстовый поиск (SQLite FTS5 → PostgreSQL `tsvector` после миграции)
- Фильтры: регион, бюджет, тип работ, срок, уровень верификации
- Сортировки: рейтинг, дата, расстояние, бюджет
- Сохранённые поиски + email-уведомления

### Реал-тайм коммуникация

- WebSocket-чат на `FastAPI WebSocket` + Redis Pub/Sub
- Доставка/прочтение, индикатор "печатает"
- Файлы и фото в чате (через S3/Yandex Object Storage)

### Отзывы и рейтинги

- Отзыв доступен **только** после `Project.status == COMPLETED`
- Двусторонний (заказчик ↔ исполнитель)
- Структурированные критерии: качество, сроки, коммуникация, цена/качество

### Smart Escrow

- Модель `Transaction` с состояниями: `PENDING`, `FUNDED`, `MILESTONE_*`, `RELEASED`, `DISPUTED`
- Поэтапная разблокировка по milestones
- Интеграция YooKassa / CloudPayments
- Диспуты: 72 часа на ответ, эскалация модератору

### Виртуальные бригады

- Объединение исполнителей в команды для крупных заказов
- Распределение долей оплаты, согласие участников
- Прозрачность для заказчика (видит состав бригады)

## Фаза 9 · Production-готовность

**Цель:** платформа готова к публичному запуску и нагрузке.

### Инфраструктура

- Миграция SQLite → PostgreSQL 16 (`asyncpg`)
- `docker-compose.yml` v2: `db`, `redis`, `backend`, `frontend`, `nginx`, `ollama`
- GitHub Actions: build → test → push image → deploy
- Облако: Selectel / Yandex Cloud / Timeweb

### Наблюдаемость

- **Sentry** на frontend и backend (ошибки + трассировки)
- **OpenTelemetry** → Grafana Cloud / Loki
- Health-чеки: `/health`, `/ready`
- Метрики бизнес-событий (регистрации, оплаты, диспуты)

### Безопасность и устойчивость

- Rate-limiting на эндпоинты входа/чата (`slowapi`)
- HTTPS-only, HSTS, secure cookies
- Регулярные бэкапы БД + загрузок
- Pen-test чек-лист (OWASP Top 10)

### E2E и нагрузка

- Playwright: критические пути (регистрация → проект → заявка → escrow)
- Locust / k6: 100 RPS — baseline, 1000 RPS — целевой потолок
- SLO: p95 < 300 мс, ошибок < 0.5 %

### Документация и юр. часть

- API-документация (Swagger + Redoc)
- Публичные офёрты: оферта на услуги, политика обработки ПД, оферта эскроу
- README с инструкциями для контрибьюторов
